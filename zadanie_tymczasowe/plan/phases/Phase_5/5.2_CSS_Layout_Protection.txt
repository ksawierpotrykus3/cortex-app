# NEXUS V2 IMPLEMENTATION - PHASE 5.2
**Moduł:** CSS Layout Protection & Virtual Windowing
**Cel 0-100%:** Utrzymanie gładkiego interfejsu renderowania w 60 FPS dla logów AI spływających w czasie rzeczywistym.

---

## 🛠️ [ZADANIA DLA AI - CODERA]

1. **Virtual Windowing (`src/frontend/components/LogViewer.tsx`)**:
   - Zastosuj `react-window` dla renderowania logów (lub równoważny moduł własny w React). Komponent dostaje tablicę z milionem elementów, a po stronie HTML renderuje WYŁĄCZNIE 20 elementów (te widoczne na monitorze).
2. **Strict CSS Boundary**:
   - Zapobiegnij przerysowaniom strony (Layout Shifts). Ustaw twarde ograniczniki wymiarowe na elementach listy: `.log-item { height: 60px; contain: strict; }`. Wymuś by grafiki CSS i rendering ignorowały wzajemne najeżdżanie, wyłączając tzw. Layout Thrashing.
3. **Throttling Frontendu**:
   - Agenci szturmują system 20-ma logami na sekunde. Odbiór eventu z Electron-Renderer musi mieć wpiętą funkcje limitującą (Throttling) rzutującą zbiór zdarzeń na interfejs nie częściej niż np. `requestAnimationFrame` albo co `100ms`, by nie odpalać pełnego przeliczenia drzewa DOM na każdy wpis z osobna.

**[WERYFIKACJA DETERMINISTYCZNA (TDD DLA 2026)]**
- **Test 1:** Symulowany zrzut 2,000 wiadomości od agentów z podłączonym profilerem Reacta. Mierzymy czasy przeliczania `commit time`. Ustalenie twardej granicy w TDD `expect(commitTime).toBeLessThan(16)` (wymóg na 60 FPS = 16.6 ms renderu). Jeśli rośnie liniowo wraz z logami – test przerywa kompilację by nie stworzyć zbugowanego interfejsu.
