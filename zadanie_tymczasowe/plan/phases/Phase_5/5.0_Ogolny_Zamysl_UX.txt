# NEXUS V2 IMPLEMENTATION - PHASE 5.0
**Moduł:** UX Automation & UI Stability - Ogólny Zamysł Renderowania
**Cel 0-100%:** Electron, React, Vite. Aplikacja nie może "chrupnąć" pod obciążeniem milionów rekordów wpompowywanych z IPC. Budujemy architekturę widoków w oparciu o sztywną wirtualizację UI i wstrzykiwanie Zod.

---

## 🛠️ [ZASADY TEORETYCZNE DLA AI]
1. **Odrzucenie Renderowania Całościowego**: Standardowy React "mapuje" elementy po tablicy do `<div>`. Wpuszczenie 100,000 wpisów logu z agentów na raz doprowadzi silnik Blink w Chromium do całkowitego "Page Unresponsive". Faza 5 wprowadza restrykcje paginacji dla IPC.
2. **Zero Poligonów Węzłowych**: Używamy natywnych Web Components Reactowych. Jeśli coś można ukryć poza widokiem ViewPortu (Virtual Scrolling) robimy to na twardo używając pakietów np. `react-window` lub taniej alternatywy z przecięciem natywnym `IntersectionObserver`.
3. **Draft Zone i Zod Guards**: Nie pozwalamy by człowiek wprowadzał złe komendy konfiguracyjne. UX dba o asercje. Zod blokuje puste stringi jeszcze zanim uderzą do procesu ZCSMAP.
