### SZCZEGÓŁY IMPLEMENTACYJNE

**NexusVFS** (`src/backend/security/vfs.ts`):
- Cienkie proxy — żadnego buforowania plików w RAM
- Format URI: `nexus://<node>/<subpath>`
- `resolve(uri)` → `path.resolve(mount.realPath, subPath)` + `path.relative` anti-traversal check
- `readFile(token, uri)` / `writeFile(token, uri, data)` / `deleteFile(token, uri)` / `stat(token, uri)` / `readdir(token, uri)` / `exists(token, uri)`
- Każda operacja przechodzi przez `checkAcl(token, uri, right)` przed dostępem do dysku

**AccessControl** (wbudowany w NexusVFS):
- `registerToken(token, nexusPath, rights[])` — jawne nadawanie praw
- `revokeToken(token)` — całkowite odwołanie dostępu
- `checkAcl(token, uri, requiredRight)` → `{ allowed: boolean, reason?: string }`
- Prefix match: `nexus://agents` pasuje do `nexus://agents/log_01`
- Polityka: **Default Deny** — brak tokenu = brak dostępu

### PARAMETRYZACJA ARCHITEKTONICZNA (STAN W PAMIĘCI RAM & I/O)

- **VFS_URI_FORMAT** = `nexus://<node>/<subpath>` — translacja przez `path.resolve(mount.realPath, subPath)` + `path.relative` anti-traversal check
- **VFS_ACL_POLICY** = Default Deny — `registerToken(token, nexusPath, rights[])` z prefix matchem i per-right check
- **VFS_MOUNT_SECURITY** = `mount()` waliduje `path.relative(workspaceBase, absolutePath) !== startsWith('..')` — zakaz mountowania poza workspace
- **VFS_RIGHTS** = `'read' | 'write' | 'delete' | 'execute'`
- **WINDOWS_NT_ACL** = Default Deny hierarchy enforcement across 9 native workspace nodes + 6 VFS mount points
- **SAB_SIZE** = `10 * 1024 * 1024` (10MB allocated via Electron Main Process)
- **META_VIEW** = `Int32Array(sab, 0, 16)` → `meta[0]` = HEAD offset, `meta[1]` = TAIL offset, `meta[2]` = LOCK_STATE, `meta[3]` = OPEN flag
- **DATA_VIEW** = `Uint8Array(sab, 64)` → Continuous lock-free circular ring buffer
- **WORKER_SLEEP_STRATEGY** = `Atomics.wait(meta, META_HEAD, lastKnownHead)` — blokada na poziomie jądra OS, zero CPU
- **TAIL_UPDATE_GUARANTEE** = `Atomics.store(meta, META_TAIL, newTail)` **dopiero po** `fs.promises.appendFile` — Exactly-Once
- **ZERO_IPC_CONTRACT** = Worker **nigdy** nie woła `parentPort.postMessage()` — jedynym sygnałem jest atomowa aktualizacja TAIL
