### SZCZEGÓŁY IMPLEMENTACYJNE

**LocalCLIRunner** (`src/backend/sandbox/hypervisor.ts`):
- `runCode(code, language)` — główne API dla agentów
- Dwie ścieżki wykonania:
  - **Ścieżka A** (Hypervisor): `startVM()` → `execInVM()` → `stopVM()` — pełna wirtualizacja
  - **Ścieżka B** (Local): spawn procesu z izolacją sieciową i timeoutem
- Network Isolation: czyszczenie zmiennych `HTTP_PROXY`, `HTTPS_PROXY`, `http_proxy`, `https_proxy` + flaga `NEXUS_NETWORK_ISOLATED=1`
- Timeout: `setTimeout` → `taskkill /F /T` (Windows) / `SIGKILL` (Linux) → reject z `ExecutionTimeoutErrorLocal`

**HypervisorProvider** (interface):
```typescript
interface HypervisorProvider {
  readonly name: string;
  startVM(config: MicroVMConfig): Promise<VMInstance>;
  stopVM(instance: VMInstance): Promise<void>;
  execInVM(instance: VMInstance, command: string, args: string[], options?: ExecutionOptions): Promise<ExecutionResult>;
  isVMRunning(instance: VMInstance): Promise<boolean>;
  listRunningVMs(): Promise<VMInstance[]>;
}
```
- Gotowość do implementacji przez natywne bindings Rust/C (libkrun-sys, whpx-sys)
- Możliwość testowania przez `mockHypervisor` (test kontraktu interfejsu)

### UWAGA IMPLEMENTACYJNA

Właściwa implementacja WHPX/libkrun wymaga natywnych bindingów C/Rust (libkrun-sys, whpx-sys), które wykraczają poza scope czystego TypeScript/Node.js. Obecna warstwa (`LocalCLIRunner`) stanowi:

1. **Abstrakcję** — interfejs `HypervisorProvider` gotowy do implementacji w Rust/C
2. **Funkcjonalne zabezpieczenie** — izolacja sieciowa, timeout, circuit breaker działają w czystym Node.js
3. **Gotowość produkcyjną** — `LocalCLIRunner.createDefaultRunner()` nadaje się do użycia z agentami AI

### PARAMETRYZACJA ARCHITEKTONICZNA (STAN W PAMIĘCI RAM & I/O)

- **MICROVM_HYPERVISOR_INTERFACE** = `HypervisorProvider`: `startVM(config)`, `stopVM(instance)`, `execInVM(instance, cmd, args)`, `isVMRunning(instance)`, `listRunningVMs()`
- **MICROVM_DEFAULT_CONFIG** = `{ vcpuCount: 1, memoryMB: 256, networkIsolated: true }`
- **LOCAL_RUNNER_TIMEOUT** = `30000ms` (domyślnie, konfigurowalny przez `LocalCLIRunner({ defaultTimeoutMs })`)
- **LOCAL_RUNNER_MAX_OUTPUT** = `1_000_000` bajtów (maksymalny stdout/stderr)
- **SAB_SIZE** = `10 * 1024 * 1024` (10MB allocated via Electron Main Process)
- **META_VIEW** = `Int32Array(sab, 0, 16)` → `meta[0]` = HEAD offset, `meta[1]` = TAIL offset, `meta[2]` = LOCK_STATE, `meta[3]` = OPEN flag
- **DATA_VIEW** = `Uint8Array(sab, 64)` → Continuous lock-free circular ring buffer
- **WORKER_SLEEP_STRATEGY** = `Atomics.wait(meta, META_HEAD, lastKnownHead)` — blokada na poziomie jądra OS, zero CPU
- **TAIL_UPDATE_GUARANTEE** = `Atomics.store(meta, META_TAIL, newTail)` **dopiero po** `fs.promises.appendFile` — Exactly-Once
- **ZERO_IPC_CONTRACT** = Worker **nigdy** nie woła `parentPort.postMessage()` — jedynym sygnałem jest atomowa aktualizacja TAIL
