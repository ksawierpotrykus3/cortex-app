### SZCZEGÓŁY IMPLEMENTACYJNE

**ExecutionRunner** (`src/backend/sandbox/execution.ts`):
- `run(command, args, stdin?)` — spawn procesu + timeout
- Mierzy czas przez `process.hrtime.bigint()` z dokładnością do nanosekund
- Timeout wykonuje:
  1. `child.kill('SIGKILL')` — szybka próba
  2. `ZombieKiller.kill(pid)` — `taskkill /F /T` (gwarancja Windows kernel)
  3. Odrzuca promise przez `ExecutionTimeoutError` z PID i elapsed time
- Po normalnym zakończeniu: Zombie check → jeśli proces nadal żyje jako zombie → `ZombieKiller.kill()`
- Obsługuje stdin/stdout/stderr z limitem `maxOutputBytes`

**ZombieKiller**:
- `kill(pid)` — taskkill z max 3 próbami
- `isProcessAlive(pid)`:
  - Windows: `tasklist /FI "PID eq N" /NH` → sprawdza czy PID istnieje w outputcie
  - Linux: `process.kill(pid, 0)` → sprawdza czy proces istnieje
- Między próbami: `200ms` odstępu (konfigurowalne)
- Po 3 nieudanych próbach: `ZombieProcessError` — wymaga interwencji administratora

**Safety**:
- Automatyczny `ZombieCheck` po normalnym zakończeniu procesu
- Idempotentne kill: jeśli proces już nie istnieje → `{ success: true }`
- Ochrona przed `pid <= 0`: natychmiastowy `{ success: false }`

### PARAMETRYZACJA ARCHITEKTONICZNA (STAN W PAMIĘCI RAM & I/O)

- **CIRCUIT_BREAKER_TIMEOUT** = `1500ms` (domyślnie, konfigurowalny przez `ExecutionRunner({ timeoutMs })`)
- **CIRCUIT_BREAKER_KILL** = Windows: `taskkill /F /T /PID <pid>`, Linux: `SIGKILL (-9)`
- **ZOMBIE_KILLER_STRATEGY** = `tasklist /FI "PID eq N" /NH` → weryfikacja → ponowny taskkill (max 3 próby, `200ms` odstęp)
- **ZOMBIE_KILLER_MAX_ATTEMPTS** = `3`
- **ZOMBIE_KILLER_CHECK_DELAY** = `200ms`
- **ZOMBIE_PROCESS_ERROR** = Tylko jeśli proces przeżyje 3 próby taskkill
- **SAB_SIZE** = `10 * 1024 * 1024` (10MB allocated via Electron Main Process)
- **META_VIEW** = `Int32Array(sab, 0, 16)` → `meta[0]` = HEAD offset, `meta[1]` = TAIL offset, `meta[2]` = LOCK_STATE, `meta[3]` = OPEN flag
- **DATA_VIEW** = `Uint8Array(sab, 64)` → Continuous lock-free circular ring buffer
- **WORKER_SLEEP_STRATEGY** = `Atomics.wait(meta, META_HEAD, lastKnownHead)` — blokada na poziomie jądra OS, zero CPU
- **TAIL_UPDATE_GUARANTEE** = `Atomics.store(meta, META_TAIL, newTail)` **dopiero po** `fs.promises.appendFile` — Exactly-Once
- **ZERO_IPC_CONTRACT** = Worker **nigdy** nie woła `parentPort.postMessage()` — jedynym sygnałem jest atomowa aktualizacja TAIL
