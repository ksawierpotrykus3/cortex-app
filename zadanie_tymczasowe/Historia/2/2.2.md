### STATUS ZGODNOŚCI ZE SPECYFIKACJĄ

- ✅ **Binary Stream (zakaz readline)**: `createReadStream` bez kodowania + skanowanie `Uint8Array` dla `0x0A` = `'\n'`
- ✅ **Detekcja nagłówków Hex**: samodzielne sprawdzanie `0x0A` w strumieniu binarnym — brak `readline`, brak string processing
- ✅ **Odporność na \r\n vs \n**: separator `0x0A`, obcięcie `0x0D` (CR) przed `JSON.parse` — identyczne działanie Windows/Linux
- ✅ **Ochrona UTF-8**: `0x0A` nie występuje w środku sekwencji wielobajtowej — znaki "ą" "ę" nigdy nie są rozcinane
- ✅ **Ochrona przed OOM**: stream 256KB chunki + `MAX_RECORD_SIZE = 10MB` — 1GB log nie zgniecie V8
- ✅ **Precyzyjne offsety**: 2 przebiegi: skan `0x0A` → lista pozycji → `handle.read(buffer, 0, len, offset)` z dokładnym offsetem
- ✅ **Tombstone detection**: `parsed.tombstone === true` → `isTombstone: true` w `DocumentMeta`

### PARAMETRYZACJA ARCHITEKTONICZNA (STAN W PAMIĘCI RAM & I/O)

- **SAB_SIZE** = `10 * 1024 * 1024` (10MB allocated via Electron Main Process)
- **META_VIEW** = `Int32Array(sab, 0, 16)` → `meta[0]` = HEAD offset, `meta[1]` = TAIL offset, `meta[2]` = LOCK_STATE, `meta[3]` = OPEN flag
- **DATA_VIEW** = `Uint8Array(sab, 64)` → Continuous lock-free circular ring buffer
- **DISK_WRITE_FORMAT (Writer)** = Native JSONL via `fs.promises.appendFile`: `{"t":<timestamp_ms>,"p":"<raw_text>"}\n`
- **DISK_WRITE_FORMAT (KeyDir)** = Native JSONL via `fs.promises.appendFile`: `{"id":"<key>","timestamp":<ms>,"data":<object>}\n` lub `{"id":"<key>","timestamp":<ms>,"tombstone":true}\n`
- **KEYDIR_RAM_INDEX** = `Map<string, DocumentMeta>` gdzie `DocumentMeta = { filePath, offset, size, timestamp, isTombstone }`
- **INDEXER_REBUILD_STRATEGY** = 2-pass: `createReadStream` (binary, 256KB chunk) → scan `0x0A` → `fs.open` + `handle.read(buffer, 0, len, offset)`
- **INDEXER_LINE_ENDING** = separator `0x0A`, trim `0x0D` przed parse — obsługa `\r\n` i `\n`
- **MAX_RECORD_SIZE** = `10 * 1024 * 1024` (ochrona przed OOM)
- **WORKER_SLEEP_STRATEGY** = `Atomics.wait(meta, META_HEAD, lastKnownHead)` — blokada na poziomie jądra OS
- **TAIL_UPDATE_GUARANTEE** = `Atomics.store(meta, META_TAIL, newTail)` **dopiero po** `fs.promises.appendFile` — Exactly-Once
- **ZERO_IPC_CONTRACT** = Worker **nigdy** nie woła `parentPort.postMessage()` — jedynym sygnałem jest atomowa aktualizacja TAIL
- **WINDOWS_NT_ACL** = Default Deny hierarchy enforcement across 9 native workspace nodes
