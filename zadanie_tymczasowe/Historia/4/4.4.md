### PARAMETRYZACJA ARCHITEKTONICZNA (STAN W PAMIĘCI RAM & I/O)

- **SAB_SIZE** = `10 * 1024 * 1024` (10MB allocated via Electron Main Process)
- **META_VIEW** = `Int32Array(sab, 0, 16)` → `meta[0]` = HEAD offset, `meta[1]` = TAIL offset, `meta[2]` = LOCK_STATE, `meta[3]` = OPEN flag
- **DATA_VIEW** = `Uint8Array(sab, 64)` → Continuous lock-free circular ring buffer
- **DISK_WRITE_FORMAT (Writer)** = Native JSONL via `fs.promises.appendFile`: `{"t":<timestamp_ms>,"p":"<raw_text>"}\n`
- **DISK_WRITE_FORMAT (KeyDir)** = Native JSONL via `fs.promises.appendFile`: `{"id":"<key>","timestamp":<ms>,"data":<object>}\n` lub `{"id":"<key>","timestamp":<ms>,"tombstone":true}\n`
- **KEYDIR_RAM_INDEX** = `Map<string, DocumentMeta>` gdzie `DocumentMeta = { filePath, offset, size, timestamp, isTombstone }`
- **COMPACT_THRESHOLD** = `10 * 1024 * 1024` (próg automatycznej kompakcji — 10MB)
- **COMPACT_STRATEGY** = fs.open → read live → fs.open .tmp → write → fs.rename → batch update KeyDir (brak okna wyścigu)
- **CIRCUIT_BREAKER_TIMEOUT** = `1500ms` (domyślnie, konfigurowalny)
- **ZOMBIE_KILLER_STRATEGY** = `tasklist /FI "PID eq N" /NH` → weryfikacja → ponowny taskkill (max 3 próby, 200ms odstęp)
- **WORKER_SLEEP_STRATEGY** = `Atomics.wait(meta, META_HEAD, lastKnownHead)` — blokada na poziomie jądra OS
- **TAIL_UPDATE_GUARANTEE** = `Atomics.store(meta, META_TAIL, newTail)` **dopiero po** `fs.promises.appendFile` — Exactly-Once
- **ZERO_IPC_CONTRACT** = Worker nigdy nie woła `parentPort.postMessage()` — jedynym sygnałem jest atomowa aktualizacja TAIL
- **WINDOWS_NT_ACL** = Default Deny hierarchy enforcement across 9 native workspace nodes
- **WATCHER_STABILITY_THRESHOLD** = `2000ms` (awaitWriteFinish stabilityThreshold)
- **WATCHER_POLL_INTERVAL** = `100ms` (awaitWriteFinish pollInterval)
- **WATCHER_IGNORED** = `[\\/]Buffers[\\/]`, `[\\/]\.tmp$`, `[\\/]\.compact\.tmp$`, `\.lock$`
- **DAG_MAX_STACK_DEPTH** = `10` (PipelineContext.MAX_STACK_DEPTH — twardy limit hopów)
- **DAG_ROUTING_STRATEGIES** = exact match ID → RegExp → string prefix
- **ONNX_SINGLETON** = `OnnxRuntime.getInstance()` — model all-MiniLM-L6-v2.onnx ładowany DOKŁADNIE RAZ
- **ONNX_EXECUTION_PROVIDER** = `['cpu']` — zero GPU, zero sieci
- **ONNX_GRAPH_OPTIMIZATION** = `'all'` (`graphOptimizationLevel`)
- **ONNX_TENSOR_DISPOSAL** = `tensor.dispose()` na KAŻDYM tensorze (input / mask / type / output) po każdym forward pass
- **ONNX_MAX_TOKENS** = `256` (maxTokens dla tokenizera)
- **ONNX_EMBEDDING_DIMS** = `384` (wymiar wektora all-MiniLM-L6-v2)
- **ONNX_TOKENIZER** = WordPiece hash: [CLS]=101 → hash słów → [SEP]=102, zakres [103, 29103]
- **ONNX_MEAN_POOLING** = dla `last_hidden_state` [1, seq_len, 384] → średnia po seq_len
- **LEXICAL_BM25_PARAMS** = `{ k: 100, k1: 1.2, b: 0.75, norm: 'l2' }`
- **LEXICAL_WINK_PIPES** = `['sbd', 'negation', 'pos', 'ner', 'sentiment']`
- **LEXICAL_NEGATION_FILTER** = dokumenty z zanegowanymi terminami są odrzucane (`hasNegatedTerm → continue`)
- **RRF_K** = `60` (stała Reciprocal Rank Fusion)
- **RRF_STRATEGY** = `Promise.allSettled` — niezależna obsługa błędów każdego silnika
- **RRF_FUSION_MAP** = `Map<docIndex, { semanticRank, lexicalRank }>` → `score = Σ 1/(k + rank)`
- **MODEL_CACHE_MIN_SIZE** = `1_000_000` (minimalny rozmiar pliku .onnx w bajtach — ~1MB)
- **MODEL_CACHE_DIR** = `models/` (w katalogu projektu, resolvany jako pierwszy istniejący)
